AppealDesk

Appeal Management Platform — Application Overview

Prepared for: MSSV & Co — Chartered Accountants
Version 1.0  |  April 2026

Introduction

What is AppealDesk?

A web-based platform that replaces Excel for tracking and managing income tax appeals across clients.

The Problem It Solves

MSSV & Co manages 1,000+ income tax appeals using spreadsheets — leading to data inconsistency, no real-time visibility, and manual coordination overhead across team members. AppealDesk centralises all appeal data in one secure, multi-user web application accessible from any browser (desktop or mobile).

Key Benefits

  • Real-time access for the entire team
  • Centralised appeal data — no more Excel silos
  • Track deadlines, hearings, and outcomes
  • Role-based access (admin, staff, client)
  • Document storage attached to each appeal
  • Complete audit trail of all actions

Who Uses It

  • Platform Admin — manages the platform itself
  • SP Admin — MSSV senior partners/admins
  • SP Staff — CA team members handling appeals
  • Client Users — client company staff (view-only)

Two Portals

The system has two distinct portals, each with its own purpose and user base:

  • Platform Portal — system-level administration
  • SP Portal — day-to-day appeal management for MSSV & clients

Access & Security

  • Email + password login (Supabase Auth)
  • Row-level security — users only see their data
  • Role-based permissions enforced server-side
  • All actions logged in an audit trail

Platform Portal

System-level administration — accessed by the Platform Super Admin only

The Platform Portal is the top-level control panel for AppealDesk. It is used to set up and govern the entire system — creating service providers, managing master data, and configuring branding. Regular firm users never access this portal.

Dashboard

High-level overview of the platform — total service providers, active firms, and system health at a glance.

Service Providers

Create and manage CA firms (service providers) that use AppealDesk. Each provider gets their own isolated workspace with their own clients, users, and appeals.

  • Add / edit / deactivate a service provider
  • Set provider name, PAN, GST, contact details
  • Upload service provider logo
  • View provider detail page

Admins

Manage platform-level administrator accounts. These are the super-admins who have full access to the Platform Portal.

  • Create / deactivate platform admin users
  • Assign admin roles

Master Records

Maintain global reference data used across all service providers, such as authority types, proceeding types, and other dropdown values.

  • Manage authority types and names
  • Manage proceeding type categories
  • Maintain jurisdiction and city lists

Settings

Configure platform-wide branding and support information that appears across the entire application, including the login page.

  • Platform Logo — upload the logo displayed on the login page and platform sidebar
  • Platform Name — the name shown throughout the application
  • Support Email — contact shown to users who need help
  • Account Settings — update platform admin's own password

Who can access:  Super Admin  Only the platform super admin account created during setup can log into the Platform Portal. All other users (SP admins, staff, clients) go through the SP Portal.

Service Provider (SP) Portal

The main workspace for MSSV & Co — where appeals are managed day-to-day

The SP Portal is where the CA firm team (admins, staff) and their clients interact with the system. This is the primary interface used every day for creating and tracking income tax appeals.

Dashboard

A live summary of the workspace. Shows key metrics at a glance:

  • Total active appeals and clients
  • Appeals by importance (Critical / High / Medium / Low)
  • Outcome distribution (Favourable / Doubtful / Unfavourable)
  • Upcoming deadlines within the next 30 days
  • Recent activity feed
  • Client users see only their organisation's appeals

Appeals (Core Module)

The central module — full detail on the next page.

  • List all appeals with filters and search
  • Create new appeals
  • View full appeal detail with proceedings and events
  • Upload and manage documents

Clients

Manage the client organisations whose appeals are handled by MSSV.

  • Add / edit / view client organisations
  • Store PAN, TAN, GST, address details
  • View all appeals belonging to a client
  • Manage client contact staff

Users

Manage all user accounts in the workspace — SP team and client users.

  • Create SP Admin, SP Staff, and Client user accounts

Documents

A consolidated view of all documents uploaded across all appeals — searchable and filterable by client or appeal.

Audit Logs Admin only

A chronological record of every significant action performed in the system.

Core Module

Appeals — In Detail

The Appeals module is the heart of AppealDesk. Each appeal represents one income tax matter for a client and contains all related proceedings, events, and documents in one place.

Appeal Structure

An Appeal is the top-level record for a tax matter (e.g., AY 2023-24, Income Tax Act). Under each appeal, there can be one or more Proceedings (e.g., Assessment, Penalty). Each proceeding contains a timeline of Events (notices, hearings, orders, etc.).

Appeal  →  Proceedings  →  Events  →  Documents

Appeal Fields

  • Client Organisation
  • Financial Year (e.g. 2023-24)
  • Assessment Year (e.g. AY 2024-25)
  • Act / Regulation (e.g. Income Tax Act)

Proceeding Fields

  • Proceeding type (Assessment, Penalty, etc.)
  • Authority type and authority name
  • Jurisdiction and city
  • Importance level (Critical / High / Medium / Low)
  • Mode (Online / Offline)
  • Initiated on & completion deadline
  • Assigned to (SP staff member)
  • Client staff contact
  • Possible outcome (Favourable / Doubtful / Unfavourable)

Event Types (Timeline entries within a Proceeding)

Event Type Description Key Fields Captured
Notice from Authority A notice received from the tax authority Date of notice, response due date, notice details
Response to Notice A reply or submission made to the authority Response submitted date, method, summary
Adjournment Request Request to postpone a hearing Original date, adjourned-to date, reason
Personal Hearing In-person hearing with the authority Hearing date, location, attendees, outcome
Virtual Hearing Online hearing / video conference Hearing date, platform, link, outcome
Personal Follow-up A follow-up visit or communication Against notice dated, follow-up details
Assessment Order Final order issued by the assessing officer Date of order, demand amount, outcome
Notice of Penalty Penalty notice received from the authority Date of notice, penalty section, amount
Penalty Order Final penalty order issued Date of order, confirmed penalty amount

What You Can Do in Appeals

  • Create a new appeal with initial proceeding
  • Edit appeal details (client, FY, AY, act)
  • Add multiple proceedings to one appeal
  • Edit proceeding details and assignment
  • Add events to the proceeding timeline
  • Quick-view or edit any event in a modal
  • Delete events (with confirmation)
  • Upload documents (PDF, Word, etc.) to an appeal
  • Download or delete attached documents
  • Delete an entire appeal (with confirmation)

Appeals List — Filters & Search

  • Search by client name or keywords
  • Filter by client organisation
  • Filter by financial year
  • Filter by importance level
  • Filter by possible outcome
  • Filter by assigned team member
  • Client users automatically see only their organisation's appeals
